What is Terraform?

Terraform is an open-source Infrastructure as Code (IaC) tool created by HashiCorp. It allows you to define, provision, and manage cloud infrastructure using a declarative configuration language called HCL (HashiCorp Configuration Language).

Instead of clicking around in a cloud console, you write code that describes what infrastructure you want, and Terraform figures out how to create it.

💡
Why Terraform?

Terraform works with 3000+ providers (AWS, Azure, GCP, Kubernetes, GitHub, Datadog, etc.). Write once, deploy anywhere — with a consistent workflow regardless of provider.

How Terraform Works

PhaseCommandWhat Happens
Write—Author .tf files describing desired infrastructure
Initterraform initDownload providers & initialize backend
Planterraform planPreview changes (no actual changes)
Applyterraform applyExecute changes to reach desired state
Destroyterraform destroyTear down all managed infrastructure

HCL Syntax Deep Dive

HCL (HashiCorp Configuration Language) is the language used to write Terraform configurations. It's designed to be both human-readable and machine-parseable.

Basic Structure

main.tf HCL 
# This is a comment

# BLOCK SYNTAX: block_type "label1" "label2" { ... }
resource "aws_instance" "web_server" {
  ami           = "ami-0c55b159cbfafe1f0"  # string value
  instance_type = "t3.micro"

  # Number value
  count = 2

  # Boolean value
  monitoring = true

  # List value
  security_groups = ["sg-12345", "sg-67890"]

  # Map value
  tags = {
    Name        = "WebServer"
    Environment = "production"
    Team        = "backend"
  }

  # Nested block
  root_block_device {
    volume_size = 20
    volume_type = "gp3"
    encrypted   = true
  }
}

String Interpolation & Expressions

expressions.tf HCL 
# Basic interpolation: ${expression}
name = "${var.project}-${var.environment}-server"

# Multi-line string (heredoc)
user_data = <<-EOT
  #!/bin/bash
  apt-get update -y
  apt-get install -y nginx
  systemctl start nginx
  echo "Hello from ${var.environment}" > /var/www/html/index.html
EOT

# Ternary / conditional expression
instance_type = var.environment == "prod" ? "t3.large" : "t3.micro"

# Reference another resource attribute
subnet_id = aws_subnet.main.id

# Reference a module output
vpc_id = module.vpc.vpc_id

# Splat expression — get all IDs from a list of resources
subnet_ids = aws_subnet.public[*].id

# For expression — transform a list
upper_names = [for name in var.names : upper(name)]

# For expression — transform a map
tagged_amis = {for id, ami in var.ami_ids : id => upper(ami) if ami != ""}

Your First Terraform Config

Let's create a simple AWS S3 bucket — a perfect first resource to understand the Terraform workflow.

main.tf HCL 
# ─── TERRAFORM SETTINGS ────────────────────────────────────────
terraform {
  required_version = ">= 1.5.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"  # ~> means >= 5.0, < 6.0
    }
  }
}

# ─── PROVIDER CONFIGURATION ────────────────────────────────────
provider "aws" {
  region = "us-east-1"
}

# ─── RESOURCE ──────────────────────────────────────────────────
resource "aws_s3_bucket" "my_bucket" {
  bucket = "my-first-terraform-bucket-12345"

  tags = {
    Name        = "MyFirstBucket"
    Environment = "learning"
    ManagedBy   = "terraform"
  }
}

# ─── OUTPUT ────────────────────────────────────────────────────
output "bucket_name" {
  description = "The name of the S3 bucket"
  value       = aws_s3_bucket.my_bucket.bucket
}

output "bucket_arn" {
  description = "The ARN of the S3 bucket"
  value       = aws_s3_bucket.my_bucket.arn
}

Run it:

$terraform init # download AWS provider
$terraform fmt # format code
$terraform validate# check syntax
$terraform plan # preview changes
$terraform apply # create the bucket
$terraform destroy# delete everything

Providers are plugins that allow Terraform to interact with APIs. Each provider exposes resources and data sources.

Configuring Multiple Providers

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.0"
    }
    google = {
      source  = "hashicorp/google"
      version = "~> 4.0"
    }
    kubernetes = {
      source  = "hashicorp/kubernetes"
      version = "~> 2.0"
    }
  }
}

provider "aws" {
  region  = "us-east-1"
  profile = "default"
}

provider "aws" {
  alias  = "west"
  region = "us-west-2"
}

  provider = aws.west

  features {}

Resources

Resources are the fundamental building blocks. Each resource block describes one or more infrastructure objects.

resources.tf — Common AWS Resources HCL 
# ─── VPC ───────────────────────────────────────────────────────
resource "aws_vpc" "main" {
  cidr_block           = "10.0.0.0/16"
  enable_dns_hostnames = true
  enable_dns_support   = true
  tags = { Name = "main-vpc" }
}

# ─── SUBNET ────────────────────────────────────────────────────
resource "aws_subnet" "public" {
  vpc_id                  = aws_vpc.main.id
  cidr_block              = "10.0.1.0/24"
  availability_zone       = "us-east-1a"
  map_public_ip_on_launch = true
  tags = { Name = "public-subnet-1a", Tier = "Public" }
}

# ─── INTERNET GATEWAY ──────────────────────────────────────────
resource "aws_internet_gateway" "igw" {
  vpc_id = aws_vpc.main.id
  tags   = { Name = "main-igw" }
}

# ─── SECURITY GROUP ────────────────────────────────────────────
resource "aws_security_group" "web" {
  name        = "web-sg"
  description = "Allow HTTP and HTTPS traffic"
  vpc_id      = aws_vpc.main.id

  ingress {
    description = "HTTP"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    description = "HTTPS"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

# ─── EC2 INSTANCE ──────────────────────────────────────────────
resource "aws_instance" "web" {
  ami                    = "ami-0c55b159cbfafe1f0"
  instance_type          = "t3.micro"
  subnet_id              = aws_subnet.public.id
  vpc_security_group_ids = [aws_security_group.web.id]
  key_name               = var.key_pair_name

  root_block_device {
    volume_size = 20
    volume_type = "gp3"
    encrypted   = true
  }

  user_data = <<-EOF
    #!/bin/bash
    yum update -y
    yum install -y httpd
    systemctl start httpd
    systemctl enable httpd
  EOF

  tags = {
    Name = "web-server"
    Role = "web"
  }
}

Variables

Variables make your configurations reusable. There are three kinds: input variables, output variables, and local values.

variables.tf HCL 
# ─── BASIC VARIABLE ────────────────────────────────────────────
variable "region" {
  description = "AWS region to deploy into"
  type        = string
  default     = "us-east-1"
}

# ─── VARIABLE WITH VALIDATION ──────────────────────────────────
variable "environment" {
  description = "Deployment environment"
  type        = string

  validation {
    condition     = contains(["dev", "staging", "prod"], var.environment)
    error_message = "Environment must be dev, staging, or prod."
  }
}

# ─── SENSITIVE VARIABLE ────────────────────────────────────────
variable "db_password" {
  description = "Database password"
  type        = string
  sensitive   = true  # won't show in plan/apply output
}

# ─── COMPLEX VARIABLE ──────────────────────────────────────────
variable "vpc_config" {
  description = "VPC configuration"
  type = object({
    cidr_block          = string
    enable_dns_support  = bool
    public_subnet_cidrs = list(string)
  })
  default = {
    cidr_block          = "10.0.0.0/16"
    enable_dns_support  = true
    public_subnet_cidrs = ["10.0.1.0/24", "10.0.2.0/24"]
  }
}

# ─── INSTANCE TYPE MAP ─────────────────────────────────────────
variable "instance_type_map" {
  type = map(string)
  default = {
    dev     = "t3.micro"
    staging = "t3.medium"
    prod    = "t3.xlarge"
  }
}

# Usage: var.instance_type_map[var.environment]

State Management

Terraform state is how Terraform knows what infrastructure it manages. The state file (terraform.tfstate) is critical — lose it and Terraform loses track of all resources.

⚠️
Never commit terraform.tfstate to git!

Add *.tfstate and *.tfstate.backup to your .gitignore. State files often contain sensitive data like passwords and private keys.

Useful State Commands

$terraform state list # list all resources in state
$terraform state show aws_instance.web # show resource details
$terraform state mv aws_instance.old aws_instance.new # rename resource
$terraform state rm aws_instance.web # remove from state (keeps real resource)
$terraform import aws_instance.web i-1234567890abcdef # import existing
$terraform refresh # sync state with real infrastructure

Writing a VPC Module

modules/vpc/main.tf HCL 
resource "aws_vpc" "this" {
  cidr_block           = var.cidr_block
  enable_dns_hostnames = true
  tags = merge(var.tags, { Name = "${var.name}-vpc" })
}

resource "aws_internet_gateway" "this" {
  vpc_id = aws_vpc.this.id
  tags   = merge(var.tags, { Name = "${var.name}-igw" })
}

resource "aws_subnet" "public" {
  count             = length(var.public_subnet_cidrs)
  vpc_id            = aws_vpc.this.id
  cidr_block        = var.public_subnet_cidrs[count.index]
  availability_zone = var.azs[count.index]
  map_public_ip_on_launch = true
  tags = merge(var.tags, { Name = "${var.name}-public-${count.index + 1}" })
}

resource "aws_subnet" "private" {
  count             = length(var.private_subnet_cidrs)
  vpc_id            = aws_vpc.this.id
  cidr_block        = var.private_subnet_cidrs[count.index]
  availability_zone = var.azs[count.index]
  tags = merge(var.tags, { Name = "${var.name}-private-${count.index + 1}" })
}

Data Sources

Data sources allow you to fetch information about existing infrastructure. They're read-only — Terraform won't manage or destroy them.

data-sources.tf HCL 
# Get latest Amazon Linux 2 AMI
data "aws_ami" "amazon_linux" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*-x86_64-gp2"]
  }
}

# Get current AWS account info
data "aws_caller_identity" "current" {}

# Get available AZs in the current region
data "aws_availability_zones" "available" {
  state = "available"
}

# Look up an existing VPC by tag
data "aws_vpc" "existing" {
  filter {
    name   = "tag:Name"
    values = ["production-vpc"]
  }
}

# Get a secret from AWS Secrets Manager
data "aws_secretsmanager_secret_version" "db_password" {
  secret_id = "prod/db/password"
}

# Get SSM Parameter
data "aws_ssm_parameter" "db_host" {
  name = "/prod/database/host"

resource "aws_instance" "app" {

Loops & Conditionals

Terraform provides several mechanisms for creating multiple resources and conditional logic.

count — Simple Repetition

count.tf HCL 
# Create 3 subnets
resource "aws_subnet" "public" {
  count             = 3
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.${count.index}.0/24"
  availability_zone = data.aws_availability_zones.available.names[count.index]

# Conditional resource (create only in prod)
resource "aws_cloudwatch_log_group" "app" {
  count             = var.environment == "prod" ? 1 : 0
  name              = "/app/logs"
  retention_in_days = 30
}

for_each — Map/Set Iteration (Preferred)

for_each.tf HCL 
# Create IAM users from a set
variable "iam_users" {
  type    = set(string)
  default = ["alice", "bob", "charlie"]
}

resource "aws_iam_user" "team" {
  for_each = var.iam_users
  name     = each.value
  tags     = { User = each.value }
}

# Create S3 buckets from a map
variable "s3_buckets" {
  type = map(object({
    versioning = bool
    region     = string
  }))
  default = {
    logs = { versioning = false, region = "us-east-1" }
    data = { versioning = true,  region = "us-west-2" }
  }
}

resource "aws_s3_bucket" "buckets" {
  for_each = var.s3_buckets
  bucket   = "my-app-${each.key}-bucket"
}

resource "aws_s3_bucket_versioning" "buckets" {
  for_each = var.s3_buckets
  bucket   = aws_s3_bucket.buckets[each.key].id

  versioning_configuration {
    status = each.value.versioning ? "Enabled" : "Suspended"
  }
}

# for expressions — build new collections
locals {
  # List all bucket names
  bucket_names = [for key, bucket in aws_s3_bucket.buckets : bucket.bucket]

  # Map of bucket key to ARN
  bucket_arns  = {for key, bucket in aws_s3_bucket.buckets : key => bucket.arn}

  # Filter — only versioned buckets
  versioned_keys = [for key, cfg in var.s3_buckets : key if cfg.versioning]
}

Local Values

Locals are named expressions you compute once and reuse across your configuration. They reduce repetition and make complex expressions readable.

locals.tf HCL 
locals {
  # Build consistent name prefix
  name_prefix = "${var.project}-${var.environment}"

  # Common tags applied to all resources
  common_tags = {
    Project     = var.project
    Environment = var.environment
    ManagedBy   = "terraform"
    Owner       = var.owner
    CreatedAt   = timestamp()
  }

  # Select instance type based on environment
  instance_type = lookup({
    dev     = "t3.micro"
    staging = "t3.medium"
    prod    = "m5.large"
  }, var.environment, "t3.micro")

  # Compute CIDR blocks dynamically
  public_subnets  = [for i in range(var.az_count) : cidrsubnet(var.vpc_cidr, 8, i)]
  private_subnets = [for i in range(var.az_count) : cidrsubnet(var.vpc_cidr, 8, i + 10)]

  # Build an IAM policy document as a string
  account_id = data.aws_caller_identity.current.account_id
  region     = data.aws_region.current.name
}

# Usage
resource "aws_instance" "app" {
  ami           = data.aws_ami.latest.id
  instance_type = local.instance_type
  tags          = merge(local.common_tags, { Name = "${local.name_prefix}-app" })
}

Dynamic Blocks

Dynamic blocks let you generate nested block configurations programmatically — perfect when the number of nested blocks depends on a variable.

dynamic-blocks.tf HCL 
variable "ingress_rules" {
  type = list(object({
    description = string
    from_port   = number
    to_port     = number
    protocol    = string
    cidr_blocks = list(string)
  }))
  default = [
    { description = "HTTP",  from_port = 80,  to_port = 80,  protocol = "tcp", cidr_blocks = ["0.0.0.0/0"] },
    { description = "HTTPS", from_port = 443, to_port = 443, protocol = "tcp", cidr_blocks = ["0.0.0.0/0"] },
    { description = "SSH",   from_port = 22,  to_port = 22,  protocol = "tcp", cidr_blocks = ["10.0.0.0/8"] },
  ]
}

resource "aws_security_group" "dynamic" {
  name   = "dynamic-sg"
  vpc_id = aws_vpc.main.id

  # Dynamic block generates one ingress block per rule
  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      description = ingress.value.description
      from_port   = ingress.value.from_port
      to_port     = ingress.value.to_port
      protocol    = ingress.value.protocol
      cidr_blocks = ingress.value.cidr_blocks
    }
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

Lifecycle Rules

Lifecycle meta-arguments control how Terraform creates, updates, and destroys resources.

lifecycle.tf HCL 
# ─── PREVENT ACCIDENTAL DESTROY ────────────────────────────────
resource "aws_db_instance" "production" {
  identifier     = "prod-db"
  engine         = "postgres"
  instance_class = "db.r5.large"

  lifecycle {
    prevent_destroy = true  # terraform destroy will error!
  }
}

# ─── CREATE BEFORE DESTROY ─────────────────────────────────────
# Creates new resource first, then destroys old — zero downtime
resource "aws_launch_template" "app" {
  name_prefix   = "app-"
  image_id      = var.ami_id
  instance_type = "t3.medium"

  lifecycle {
    create_before_destroy = true
  }
}

# ─── IGNORE CHANGES ─────────────────────────────────────────────
# Ignore changes made outside Terraform (manual changes)
resource "aws_instance" "app" {
  ami           = var.ami_id
  instance_type = "t3.medium"

  lifecycle {
    ignore_changes = [
      ami,             # ignore AMI changes (e.g., patching done externally)
      tags["LastPatch"] # ignore a specific tag
    ]
  }
}

# ─── REPLACE TRIGGERED BY ───────────────────────────────────────
# Force recreation when another resource changes
resource "aws_instance" "web" {
  ami           = data.aws_ami.latest.id
  instance_type = "t3.micro"

  lifecycle {
    replace_triggered_by = [
      aws_launch_template.app.latest_version
    ]
  }
}

# ─── PRECONDITION / POSTCONDITION (Terraform >= 1.2) ───────────
resource "aws_instance" "validated" {
  ami           = var.ami_id
  instance_type = var.instance_type

  lifecycle {
    precondition {
      condition     = startswith(var.instance_type, "t3.") || startswith(var.instance_type, "m5.")
      error_message = "Only t3 and m5 instance families are approved."
    }
    postcondition {
      condition     = self.public_ip != ""
      error_message = "Instance must have a public IP."
    }
  }
}

  subnet     = cidrsubnet("10.0.0.0/16", 8, 1)  # "10.0.1.0/24"

Provisioners

⚠️
Use provisioners as a last resort

Prefer cloud-init, user_data, or configuration management tools (Ansible, Chef) over provisioners. Provisioners make state unpredictable and are hard to debug.

provisioners.tf HCL 
resource "aws_instance" "web" {
  ami           = data.aws_ami.amazon_linux.id
  instance_type = "t3.micro"
  key_name      = var.key_pair_name

  # Copy file to remote instance
  provisioner "file" {
    source      = "scripts/setup.sh"
    destination = "/tmp/setup.sh"

    connection {
      type        = "ssh"
      user        = "ec2-user"
      private_key = file(var.private_key_path)
      host        = self.public_ip
    }
  }

  # Run commands remotely (after creation)
  provisioner "remote-exec" {
    inline = [
      "chmod +x /tmp/setup.sh",
      "/tmp/setup.sh",
    ]
    connection {
      type        = "ssh"
      user        = "ec2-user"
      private_key = file(var.private_key_path)
      host        = self.public_ip
    }
  }

  # Run commands locally (when resource is destroyed)
  provisioner "local-exec" {
    when    = destroy
    command = "echo 'Destroying ${self.id}' >> destroyed.log"
  }
}

    steps:
      - uses: actions/checkout@v4

      - name: Setup Terraform

        with:
          terraform_version: ${{ env.TF_VERSION }}

      - name: Terraform Init
        run: terraform init

      - name: Terraform Format Check
        run: terraform fmt -check -recursive

      - name: Terraform Validate
        run: terraform validate

      - name: Terraform Plan
        run: terraform plan -no-color -out=tfplan
        if: github.event_name == 'pull_request'

      - name: Terraform Apply
        run: terraform apply -auto-approve tfplan
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'

Security Checklist

  • Never hardcode credentials — use IAM roles, env vars, or secrets managers
  • Mark sensitive variables and outputs with sensitive = true
  • Use remote backends with encryption enabled
  • Enable state locking with DynamoDB
  • Use minimum IAM permissions for the Terraform execution role
  • Scan Terraform configs with tfsec, checkov, or Snyk
  • Use prevent_destroy = true on stateful resources (RDS, S3)
  • Encrypt all EBS volumes and RDS instances at rest
  • Never commit .tfstate, .tfvars (with secrets), or .terraform/ to git

Performance Tips

  • Use -target sparingly — use separate state files for large infrastructure
  • Prefer for_each over count to avoid resource replacement on list changes
  • Use terraform plan -parallelism=20 for large deploys (default is 10)
  • Split large monolithic configs into multiple state files per component
  • Use terraform_remote_state data sources to share outputs between stacks
  • Use moved blocks instead of destroy+recreate when refactoring

The moved Block (Terraform >= 1.1)

moved.tf HCL 
# Rename a resource without destroying it
moved {
  from = aws_instance.web
  to   = aws_instance.web_server
}

# Move resource into a module
moved {
  from = aws_vpc.main
  to   = module.vpc.aws_vpc.this
}

# Move from count to for_each
moved {
  from = aws_subnet.public[0]
  to   = aws_subnet.public["us-east-1a"]
}

patterns.tf — Quick Reference HCL 
# ─── CREATE N RESOURCES ────────────────────────────────────────
resource "aws_subnet" "public" {
  count      = 3
  cidr_block = cidrsubnet(var.vpc_cidr, 8, count.index)
}

# ─── CONDITIONAL RESOURCE ──────────────────────────────────────
resource "aws_eip" "nat" {
  count = var.enable_nat_gateway ? 1 : 0
}

# ─── RESOURCE FROM MAP ─────────────────────────────────────────
resource "aws_s3_bucket" "this" {
  for_each = toset(["logs", "backups", "assets"])
  bucket   = "${var.project}-${each.key}"
}

# ─── REFERENCE ANOTHER RESOURCE ────────────────────────────────
# resource_type.resource_name.attribute
vpc_id = aws_vpc.main.id

# ─── REFERENCE A MODULE OUTPUT ─────────────────────────────────
# module.module_name.output_name
vpc_id = module.vpc.vpc_id

# ─── REFERENCE A DATA SOURCE ───────────────────────────────────
# data.data_type.data_name.attribute
ami = data.aws_ami.latest.id

# ─── CURRENT RESOURCE IN PROVISIONER ──────────────────────────
host = self.public_ip

# ─── DYNAMIC CIDR SUBNETS ──────────────────────────────────────
subnets = [for i in range(3) : cidrsubnet("10.0.0.0/16", 8, i)]
# → ["10.0.0.0/24", "10.0.1.0/24", "10.0.2.0/24"]

# ─── MERGE TAGS ────────────────────────────────────────────────
tags = merge(
  local.common_tags,
  { Name = "specific-resource-name" }
)

# ─── TEMPLATE FILE ─────────────────────────────────────────────
user_data = templatefile("${path.module}/userdata.tpl", {
  app_port = var.app_port
  db_host  = aws_db_instance.main.address
})

# ─── PATH REFERENCES ───────────────────────────────────────────
# path.module  — directory of the current module
# path.root    — directory of the root module
# path.cwd     — current working directory

# ─── DEPEND EXPLICITLY ─────────────────────────────────────────
resource "aws_db_instance" "main" {
  depends_on = [aws_security_group.db, aws_subnet.private]
}
