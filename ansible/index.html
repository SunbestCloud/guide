What Is Ansible?

The agentless automation engine every DevOps engineer needs

Ansible is an open-source IT automation tool developed by Red Hat. It enables you to automate provisioning, configuration management, application deployment, orchestration, and many other IT tasks. It uses YAML (a human-readable data serialization language) to describe automation jobs, called playbooks.

Why Ansible over other tools?

🚫Agentless

No agent software required on managed nodes. Uses SSH (Linux) or WinRM (Windows).

📖Human-readable

YAML playbooks are easy to read, write, and understand — no DSL to learn.

⚖️Idempotent

Running a playbook multiple times produces the same result. Safe to re-run anytime.

🔌3000+ Modules

Manage files, packages, services, clouds (AWS/GCP/Azure), databases, and more.

🏗️Ansible Galaxy

Thousands of community roles and collections — don't reinvent the wheel.

🔒Vault

Built-in secret management with AES-256 encryption for sensitive data.

Ansible Architecture

Control node, managed nodes, and how they connect

Ansible's architecture is simple: a control node connects over SSH to one or more managed nodes and executes tasks. No agent, no daemon — just SSH.

architecture-overview.txt ASCII 

  ┌─────────────────────────────────────────────────────────┐
  │                    CONTROL NODE                         │
  │                                                         │
  │   ansible.cfg    inventory/      playbooks/             │
  │   roles/         collections/    group_vars/            │
  │                                                         │
  │         [ Ansible Engine (Python) ]                     │
  └──────────┬────────────┬───────────────┬─────────────────┘
             │ SSH        │ SSH           │ SSH
             ▼            ▼               ▼
    ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
    │  Managed     │ │  Managed     │ │  Managed     │
    │  Node #1     │ │  Node #2     │ │  Node #3     │
    │  webserver   │ │  dbserver    │ │  appserver   │
    └──────────────┘ └──────────────┘ └──────────────┘

Core Components

📋 Inventory

A list of managed hosts, in INI or YAML format. Can be static files or dynamic scripts.

📜 Playbooks

YAML files describing the desired state. Contain one or more plays targeting host groups.

🔧 Modules

Units of work: apt, copy, service, template, file, etc.

🔌 Plugins

Extend Ansible: connection, callback, filter, lookup, and more.

📦 Roles

Reusable, structured playbook units. Standard directory layout for sharing.

Inventory Files

Telling Ansible which hosts to manage

The inventory is Ansible's source of truth about what hosts exist. It can be a simple INI file, YAML, or a dynamic script that queries your cloud provider.

INI Format (Most Common)

inventory/hosts.iniINI
# Single host
mail.example.com

# Group of web servers
[webservers]
web1.example.com
web2.example.com
web3 ansible_host=192.168.1.13 ansible_port=2222

# Group of DB servers
[dbservers]
db1.example.com ansible_user=ubuntu
db2.example.com ansible_user=ec2-user

# Groups can contain other groups
[production:children]
webservers
dbservers

# Variables for a group
[webservers:vars]
http_port=80
max_requests=1000
ansible_python_interpreter=/usr/bin/python3

YAML Format

inventory/hosts.ymlYAML
all:
  children:
    webservers:
      hosts:
        web1.example.com:
          ansible_host: 192.168.1.10
          http_port: 80
        web2.example.com:
          ansible_host: 192.168.1.11
    dbservers:
      hosts:
        db1.example.com:
          ansible_host: 192.168.1.20
          ansible_user: ubuntu
  vars:
    ansible_user: ansible
    ansible_ssh_private_key_file: ~/.ssh/ansible_key

Ad-Hoc Commands

Run one-off tasks without writing a playbook

Ad-hoc commands let you execute a single task against hosts directly from the command line. Great for quick checks and one-off operations.

Syntax: ansible [pattern] -m [module] -a "[arguments]"

Common Ad-Hoc Examples

ad-hoc-examples.shbash
# ── Connectivity ──────────────────────────────────────
ansible all -m ping

# ── Gather Facts ──────────────────────────────────────
ansible web1 -m gather_facts
ansible web1 -m setup -a "filter=ansible_distribution*"

# ── Run Shell Commands ────────────────────────────────
ansible webservers -m shell -a "uptime"
ansible webservers -m command -a "df -h"
ansible all -m shell -a "free -m | grep Mem"

# ── File Operations ───────────────────────────────────
ansible webservers -m copy -a "src=/etc/hosts dest=/tmp/hosts"
ansible webservers -m file -a "path=/tmp/test state=touch"
ansible webservers -m file -a "path=/tmp/test state=absent"

# ── Package Management ────────────────────────────────
ansible webservers -m apt  -a "name=nginx state=present" --become
ansible webservers -m yum  -a "name=nginx state=latest"  --become
ansible webservers -m apt  -a "name=nginx state=absent"  --become

# ── Service Management ────────────────────────────────
ansible webservers -m service -a "name=nginx state=started"  --become
ansible webservers -m service -a "name=nginx state=restarted" --become
ansible webservers -m service -a "name=nginx enabled=yes"    --become

# ── User Management ───────────────────────────────────
ansible all -m user -a "name=deploy state=present shell=/bin/bash" --become
ansible all -m user -a "name=deploy state=absent remove=yes" --become

# ── Transfer and Extract ──────────────────────────────
ansible webservers -m unarchive -a \
  "src=/tmp/app.tar.gz dest=/var/www/app remote_src=yes" --become

# ── Parallel execution with forks ─────────────────────
ansible webservers -m ping -f 20    # 20 parallel connections

Playbooks — Introduction

YAML automation scripts: the heart of Ansible

A playbook is a YAML file with one or more plays. Each play maps a set of tasks to a group of hosts. Tasks call modules to do the actual work.

Playbook Structure

structure-explained.ymlYAML
---  # YAML document start
# A playbook is a list of plays
- name: First play — configure web servers
  hosts: webservers          # target host group
  become: true              # use sudo/privilege escalation
  gather_facts: true        # collect host info (default)

  vars:                       # play-level variables
    app_port: 8080

  tasks:                     # ordered list of tasks
    - name: Install nginx    # descriptive task name
      ansible.builtin.apt:  # fully qualified module name
        name: nginx
        state: present
        update_cache: true

    - name: Start nginx service
      ansible.builtin.service:
        name: nginx
        state: started
        enabled: true

- name: Second play — configure DB servers
  hosts: dbservers
  become: true
  tasks:
    - name: Install MySQL
      ansible.builtin.apt:
        name: mysql-server
        state: present

Running Playbooks

run-playbook.shbash
# Basic run
ansible-playbook playbook.yml

# Specify inventory
ansible-playbook -i inventory/hosts.ini playbook.yml

# Dry run (check mode — no changes made)
ansible-playbook playbook.yml --check

# Dry run + show diffs
ansible-playbook playbook.yml --check --diff

# Verbose output (up to -vvvv)
ansible-playbook playbook.yml -v
ansible-playbook playbook.yml -vvv

# Limit to specific host or group
ansible-playbook playbook.yml --limit web1
ansible-playbook playbook.yml --limit webservers

# Pass extra variables on command line
ansible-playbook playbook.yml -e "app_version=2.1.0 env=production"

# Ask for sudo password
ansible-playbook playbook.yml --ask-become-pass

# Start at a specific task
ansible-playbook playbook.yml --start-at-task "Install nginx"

# Step through tasks one by one
ansible-playbook playbook.yml --step

Core Modules

The building blocks of every playbook

File & Copy Modules

file-modules.ymlYAML
- name: File and copy operations
  hosts: all
  become: true
  tasks:

    - name: Create a directory
      ansible.builtin.file:
        path: /opt/myapp
        state: directory
        owner: www-data
        group: www-data
        mode: '0755'

    - name: Copy a file from control node
      ansible.builtin.copy:
        src: files/nginx.conf
        dest: /etc/nginx/nginx.conf
        owner: root
        group: root
        mode: '0644'
        backup: true

    - name: Write inline content to file
      ansible.builtin.copy:
        content: |
          Hello from Ansible!
          Managed by automation.
        dest: /etc/motd

    - name: Fetch a file from remote host
      ansible.builtin.fetch:
        src: /var/log/nginx/error.log
        dest: logs/{{ inventory_hostname }}/
        flat: false

    - name: Add a line to a file (idempotent)
      ansible.builtin.lineinfile:
        path: /etc/hosts
        line: '10.0.0.5 appserver'
        state: present

    - name: Replace text using regex
      ansible.builtin.replace:
        path: /etc/nginx/nginx.conf
        regexp: 'worker_processes\s+\d+'
        replace: 'worker_processes 4'

    - name: Unarchive a tarball
      ansible.builtin.unarchive:
        src: files/app-v2.tar.gz
        dest: /opt/myapp
        owner: www-data

Package & Service Modules

package-service.ymlYAML
- name: Package and service management
  hosts: webservers
  become: true
  tasks:

    - name: Install multiple packages (apt)
      ansible.builtin.apt:
        name:
          - nginx
          - curl
          - git
          - vim
        state: present
        update_cache: true
        cache_valid_time: 3600  # only update if > 1hr old

    - name: Install package on any distro (generic)
      ansible.builtin.package:
        name: git
        state: present

    - name: Upgrade all packages
      ansible.builtin.apt:
        upgrade: dist
        update_cache: true

    - name: Ensure nginx is started and enabled
      ansible.builtin.service:
        name: nginx
        state: started
        enabled: true

    - name: Restart a service
      ansible.builtin.service:
        name: nginx
        state: restarted

    - name: Install pip package
      ansible.builtin.pip:
        name: flask
        version: '2.3.0'
        virtualenv: /opt/myapp/venv

User & Group Modules

Automated Web Server Setup

Deploy a fully configured Nginx web server with a custom homepage, firewall rules, and SSL-ready structure — from a single playbook.

nginx ufw templates handlers
project1-webserver.ymlYAML
---
- name: Deploy and configure Nginx web server
  hosts: webservers
  become: true
  vars:
    server_name: "{{ inventory_hostname }}"
    web_root: /var/www/html
    http_port: 80
    app_title: "My Ansible-Deployed Site"

  handlers:
    - name: Restart nginx
      ansible.builtin.service:
        name: nginx
        state: restarted

    - name: Reload nginx
      ansible.builtin.service:
        name: nginx
        state: reloaded

  tasks:
    - name: Update apt cache
      ansible.builtin.apt:
        update_cache: true
        cache_valid_time: 3600

    - name: Install required packages
      ansible.builtin.apt:
        name:
          - nginx
          - ufw
          - curl
        state: present

    - name: Configure UFW — allow SSH
      community.general.ufw:
        rule: allow
        name: OpenSSH

    - name: Configure UFW — allow HTTP
      community.general.ufw:
        rule: allow
        port: "{{ http_port }}"
        proto: tcp

    - name: Enable UFW
      community.general.ufw:
        state: enabled
        policy: deny

    - name: Create web root directory
      ansible.builtin.file:
        path: "{{ web_root }}"
        state: directory
        owner: www-data
        group: www-data
        mode: '0755'

    - name: Deploy custom homepage
      ansible.builtin.copy:
        content: |
          <!DOCTYPE html>
          <html><head><title>{{ app_title }}</title></head>
          <body>
            <h1>{{ app_title }}</h1>
            <p>Deployed by Ansible on {{ ansible_date_time.iso8601 }}</p>
            <p>Server: {{ inventory_hostname }} ({{ ansible_default_ipv4.address }})</p>
          </body></html>
        dest: "{{ web_root }}/index.html"
        owner: www-data
        mode: '0644'
      notify: Reload nginx

    - name: Configure nginx virtual host
      ansible.builtin.copy:
        content: |
          server {
              listen {{ http_port }};
              server_name {{ server_name }};
              root {{ web_root }};
              index index.html;
              location / {
                  try_files $uri $uri/ =404;
              }
              access_log /var/log/nginx/{{ server_name }}_access.log;
              error_log  /var/log/nginx/{{ server_name }}_error.log;
          }
        dest: /etc/nginx/sites-available/mysite
      notify: Restart nginx

    - name: Enable nginx site
      ansible.builtin.file:
        src: /etc/nginx/sites-available/mysite
        dest: /etc/nginx/sites-enabled/mysite
        state: link
      notify: Restart nginx

    - name: Remove default nginx site
      ansible.builtin.file:
        path: /etc/nginx/sites-enabled/default
        state: absent
      notify: Restart nginx

    - name: Ensure nginx is started and enabled
      ansible.builtin.service:
        name: nginx
        state: started
        enabled: true

    - name: Test nginx is serving content
      ansible.builtin.uri:
        url: "http://{{ ansible_default_ipv4.address }}"
        status_code: 200
      register: result

    - name: Show result
      ansible.builtin.debug:
        msg: "✅ Web server responding: {{ result.status }}"

User Management & SSH Hardening

Create multiple users with proper groups, deploy SSH keys, and harden SSH configuration across all servers.

users ssh security sudo
project2-users.ymlYAML
---
- name: User management and SSH hardening
  hosts: all
  become: true
  vars:
    admin_users:
      - name: alice
        comment: "Alice Admin"
        groups: "sudo,developers"
        ssh_key: "ssh-ed25519 AAAAC3NzaC... alice@laptop"
      - name: bob
        comment: "Bob Dev"
        groups: "developers"
        ssh_key: "ssh-ed25519 AAAAC3NzaC... bob@laptop"
    removed_users:
      - olduser
      - testaccount

  handlers:
    - name: Restart sshd
      ansible.builtin.service:
        name: sshd
        state: restarted

  tasks:
    - name: Create developers group
      ansible.builtin.group:
        name: developers
        state: present

    - name: Create admin users
      ansible.builtin.user:
        name: "{{ item.name }}"
        comment: "{{ item.comment }}"
        groups: "{{ item.groups }}"
        shell: /bin/bash
        create_home: true
        state: present
      loop: "{{ admin_users }}"

    - name: Deploy SSH authorized keys
      ansible.posix.authorized_key:
        user: "{{ item.name }}"
        key: "{{ item.ssh_key }}"
        state: present
        exclusive: true  # remove other keys
      loop: "{{ admin_users }}"

    - name: Configure sudoers for sudo group
      ansible.builtin.copy:
        content: "%sudo ALL=(ALL:ALL) NOPASSWD: ALL\n"
        dest: /etc/sudoers.d/sudo-group
        validate: /usr/sbin/visudo -cf %s
        mode: '0440'

    - name: Remove old/unwanted users
      ansible.builtin.user:
        name: "{{ item }}"
        state: absent
        remove: true
      loop: "{{ removed_users }}"

    - name: Harden SSH configuration
      ansible.builtin.blockinfile:
        path: /etc/ssh/sshd_config
        block: |
          PermitRootLogin no
          PasswordAuthentication no
          PubkeyAuthentication yes
          MaxAuthTries 3
          X11Forwarding no
          AllowTcpForwarding no
          ClientAliveInterval 300
          ClientAliveCountMax 2
        backup: true
      notify: Restart sshd

Jinja2 Templates

Generate dynamic configuration files from variables

Jinja2 is the templating engine Ansible uses. Templates (.j2 files) contain placeholders that get replaced with actual variable values when deployed.

Nginx Config Template

templates/nginx.conf.j2Jinja2

#!/usr/bin/python3
"""Custom Ansible module: manage application configuration."""

from ansible.module_utils.basic import AnsibleModule
import json, os

DOCUMENTATION = r"""
---
module: my_app_config
short_description: Manage app JSON config file
options:
  path:
    description: Path to config file
    required: true
  key:
    description: Config key
    required: true
  value:
    description: Config value
    required: true
  state:
    description: present or absent
    default: present
"""

def run_module():
    module_args = dict(
        path=dict(type='str', required=True),
        key=dict(type='str', required=True),
        value=dict(type='str', required=False),
        state=dict(type='str', default='present', choices=['present', 'absent']),
    )
    result = dict(changed=False, message='')
    module = AnsibleModule(argument_spec=module_args, supports_check_mode=True)

    path  = module.params['path']
    key   = module.params['key']
    value = module.params['value']
    state = module.params['state']

    config = {}
    if os.path.exists(path):
        with open(path, 'r') as f:
            config = json.load(f)

    if state == 'present':
        if config.get(key) != value:
            config[key] = value
            result['changed'] = True
            result['message'] = f'Set {key}={value}'
    elif state == 'absent':
        if key in config:
            del config[key]
            result['changed'] = True
            result['message'] = f'Removed {key}'

    if result['changed'] and not module.check_mode:
        with open(path, 'w') as f:
            json.dump(config, f, indent=2)

    module.exit_json(**result)

if __name__ == '__main__':
    run_module()
