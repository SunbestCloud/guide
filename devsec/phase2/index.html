🏗️ What Is IaC Security and Why Does It Change Everything?

You've been provisioning AWS infrastructure manually — clicking through the console or running CLI commands. That works, but it doesn't scale and it doesn't audit. Infrastructure as Code means your entire AWS environment — VPCs, security groups, EC2 instances, IAM roles, S3 buckets — is defined in version-controlled files.

The security superpower: If your infrastructure is code, it can be scanned like code. Before you run terraform apply and create a real S3 bucket with public access, Checkov can catch it in your CI pipeline and block the merge. Before you push a security group that opens port 22 to 0.0.0.0/0, tfsec flags it.

Real numbers: The Capital One breach (2019, $80M fine) was caused by an overly permissive IAM role on an EC2 instance — the exact type of misconfiguration that Checkov catches in under 1 second. Every project below teaches you how to prevent exactly these scenarios.

Provider & Backend Configuration

Critical security point: Store Terraform state in S3 with encryption and locking — never commit .tfstate files to git. They contain sensitive resource data.

hcl — environments/dev/backend.tf
# Remote state — encrypted, locked, versioned
terraform {
  backend "s3" {
    bucket         = "myorg-terraform-state-dev"
    key            = "vpc/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true           # AES-256 encryption at rest
    dynamodb_table = "terraform-state-lock"  # Prevents concurrent runs
    kms_key_id     = "arn:aws:kms:us-east-1:123456789:key/xxx"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }

}

VPC Core — main.tf

Build the VPC with proper CIDR planning, DNS settings, and a 3-tier subnet architecture: public (ALB), private (EC2/EKS), and isolated (RDS/ElastiCache).

hcl — modules/vpc/main.tf
# ═══ VPC ════════════════════════════════════════════════
resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true   # Required for EKS, RDS endpoints
  enable_dns_support   = true

  tags = { Name = "${var.name}-vpc" }
}

# ═══ INTERNET GATEWAY ═══════════════════════════════════
resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id
  tags   = { Name = "${var.name}-igw" }
}

# ═══ PUBLIC SUBNETS (ALB, NAT GW live here) ═════════════
resource "aws_subnet" "public" {
  count             = length(var.availability_zones)
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 8, count.index)
  availability_zone = var.availability_zones[count.index]

  # ✅ Do NOT auto-assign public IPs — assign explicitly when needed
  map_public_ip_on_launch = false

  tags = { Name = "${var.name}-public-${count.index + 1}", Tier = "public" }
}

# ═══ PRIVATE SUBNETS (EC2, EKS nodes) ══════════════════
resource "aws_subnet" "private" {
  count             = length(var.availability_zones)
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 8, count.index + 10)
  availability_zone = var.availability_zones[count.index]

  map_public_ip_on_launch = false   # Never public

  tags = { Name = "${var.name}-private-${count.index + 1}", Tier = "private" }
}

# ═══ ISOLATED SUBNETS (RDS, ElastiCache — NO internet) ══
resource "aws_subnet" "isolated" {
  count             = length(var.availability_zones)
  vpc_id            = aws_vpc.main.id
  cidr_block        = cidrsubnet(var.vpc_cidr, 8, count.index + 20)
  availability_zone = var.availability_zones[count.index]

  map_public_ip_on_launch = false

  tags = { Name = "${var.name}-isolated-${count.index + 1}", Tier = "isolated" }
}

# ═══ ELASTIC IPs FOR NAT GATEWAYS ════════════════════════
resource "aws_eip" "nat" {
  count  = length(var.availability_zones)
  domain = "vpc"
  tags   = { Name = "${var.name}-nat-eip-${count.index + 1}" }
}

# ═══ NAT GATEWAYS (one per AZ for HA) ═══════════════════
resource "aws_nat_gateway" "main" {
  count         = length(var.availability_zones)
  allocation_id = aws_eip.nat[count.index].id
  subnet_id     = aws_subnet.public[count.index].id
  depends_on    = [aws_internet_gateway.main]

  tags = { Name = "${var.name}-nat-${count.index + 1}" }
}

# ═══ ROUTE TABLES ════════════════════════════════════════
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id
  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main.id
  }
  tags = { Name = "${var.name}-public-rt" }
}

resource "aws_route_table" "private" {
  count  = length(var.availability_zones)
  vpc_id = aws_vpc.main.id
  route {
    cidr_block     = "0.0.0.0/0"
    nat_gateway_id = aws_nat_gateway.main[count.index].id
  }
  tags = { Name = "${var.name}-private-rt-${count.index + 1}" }
}

# Isolated subnets have NO default route — no internet at all
resource "aws_route_table" "isolated" {
  vpc_id = aws_vpc.main.id
  tags   = { Name = "${var.name}-isolated-rt" }
}

# Route table associations
resource "aws_route_table_association" "public" {
  count          = length(var.availability_zones)
  subnet_id      = aws_subnet.public[count.index].id
  route_table_id = aws_route_table.public.id
}

resource "aws_route_table_association" "private" {
  count          = length(var.availability_zones)
  subnet_id      = aws_subnet.private[count.index].id
  route_table_id = aws_route_table.private[count.index].id
}

resource "aws_route_table_association" "isolated" {
  count          = length(var.availability_zones)
  subnet_id      = aws_subnet.isolated[count.index].id
  route_table_id = aws_route_table.isolated.id
}

VPC Flow Logs — Critical for Security Investigation

Flow logs capture all network traffic metadata. Essential for detecting port scanning, data exfiltration, and lateral movement. Without them, you're blind during an incident.

hcl — VPC Flow Logs
# CloudWatch Log Group for flow logs
resource "aws_cloudwatch_log_group" "flow_logs" {
  name              = "/aws/vpc/${var.name}/flow-logs"
  retention_in_days = 90    # Keep 90 days minimum for compliance
  kms_key_id        = var.kms_key_arn   # Encrypt logs at rest
}

# IAM role for VPC to write to CloudWatch
resource "aws_iam_role" "flow_logs" {
  name = "${var.name}-vpc-flow-logs"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Action    = "sts:AssumeRole"
      Effect    = "Allow"
      Principal = { Service = "vpc-flow-logs.amazonaws.com" }
    }]
  })
}

resource "aws_iam_role_policy" "flow_logs" {
  name = "vpc-flow-logs-policy"
  role = aws_iam_role.flow_logs.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Action = [
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents",
        "logs:DescribeLogGroups",
        "logs:DescribeLogStreams"
      ]
      Resource = "*"
    }]
  })
}

# VPC Flow Log — capture ALL traffic (ACCEPT + REJECT)
resource "aws_flow_log" "main" {
  iam_role_arn    = aws_iam_role.flow_logs.arn
  log_destination = aws_cloudwatch_log_group.flow_logs.arn
  traffic_type    = "ALL"    # ACCEPT, REJECT, or ALL
  vpc_id          = aws_vpc.main.id

  tags = { Name = "${var.name}-flow-logs" }
}

Security Groups — Least Privilege Networking

hcl — Security Groups
# ── Application Load Balancer SG ─────────────────────
resource "aws_security_group" "alb" {
  name        = "${var.name}-alb-sg"
  description = "ALB: HTTPS only from internet"
  vpc_id      = aws_vpc.main.id

  ingress {
    description = "HTTPS from internet"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  # ❌ NEVER add port 80 here — redirect via ALB listener rule instead

  egress {
    description     = "To app layer only"
    from_port       = 8080
    to_port         = 8080
    protocol        = "tcp"
    security_groups = [aws_security_group.app.id]
  }

  tags = { Name = "${var.name}-alb-sg" }
}

# ── Application (EC2) SG ─────────────────────────────
resource "aws_security_group" "app" {
  name        = "${var.name}-app-sg"
  description = "App: traffic only from ALB, no direct internet"
  vpc_id      = aws_vpc.main.id

  ingress {
    description     = "From ALB only"
    from_port       = 8080
    to_port         = 8080
    protocol        = "tcp"
    security_groups = [aws_security_group.alb.id]
  }

  # ❌ NO SSH from internet — use AWS SSM Session Manager instead

  egress {
    description     = "To database only"
    from_port       = 5432
    to_port         = 5432
    protocol        = "tcp"
    security_groups = [aws_security_group.db.id]
  }

  egress {
    description = "HTTPS to internet via NAT (for package downloads)"
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = { Name = "${var.name}-app-sg" }
}

# ── Database SG ──────────────────────────────────────
resource "aws_security_group" "db" {
  name        = "${var.name}-db-sg"
  description = "DB: PostgreSQL from app tier only. No egress."
  vpc_id      = aws_vpc.main.id

  ingress {
    description     = "PostgreSQL from app only"
    from_port       = 5432
    to_port         = 5432
    protocol        = "tcp"
    security_groups = [aws_security_group.app.id]
  }

  # No egress rules = no outbound traffic (implicit deny)
  tags = { Name = "${var.name}-db-sg" }
}
⚠️

Golden Rule: Never use cidr_blocks = ["0.0.0.0/0"] for ingress on any port other than 80/443 on the ALB. Always reference security group IDs instead of CIDRs for internal traffic. This is one of the most common misconfigurations Checkov catches.

Secure IAM Roles & Policies as Code

Define all IAM roles, policies, and trust relationships in Terraform. Enforce least privilege, MFA conditions, service control policies, and auto-detect privilege escalation paths before deployment.

Terraform IAM Least Privilege MFA Enforcement SCPs IRSA (EKS)
◆ INTERMEDIATE ⏱ 5–7 hours
▼
ℹ️

IAM is the blast radius controller. A misconfigured IAM role is the #1 cause of AWS breaches. If your EC2 instance's IAM role has *:* permissions, one SSRF vulnerability gives an attacker access to your entire AWS account. Every permission must be justified and minimal.

1

The Dangerous Pattern vs The Secure Pattern

hcl — ❌ DANGEROUS IAM (what most people write)
# This is found everywhere — and it's a disaster waiting to happen
resource "aws_iam_role_policy" "bad" {
  policy = jsonencode({
    Statement = [{
      Effect   = "Allow"
      Action   = "*"        # ❌ Wildcard action
      Resource = "*"        # ❌ Wildcard resource
    }]
  })
}

# This gives the role FULL AWS ACCOUNT ACCESS
# One compromised EC2 = attacker controls everything
hcl — ✅ SECURE IAM — EC2 role for S3 read-only access
# Instance role — only what the app actually needs
resource "aws_iam_role" "app_role" {
  name = "${var.name}-app-role"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Allow"
      Principal = { Service = "ec2.amazonaws.com" }
      Action    = "sts:AssumeRole"
      # Condition: only from your VPC endpoints
      Condition = {
        StringEquals = {
          "aws:SourceVpc" = [var.vpc_id]
        }
      }
    }]
  })
}

resource "aws_iam_policy" "app_s3_read" {
  name        = "${var.name}-app-s3-read"
  description = "App: read-only access to specific S3 prefix"

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Effect = "Allow"
        Action = [
          "s3:GetObject",      # Read files
          "s3:ListBucket"       # List files (needed for GetObject)
        ]
        Resource = [
          "arn:aws:s3:::${var.bucket_name}",
          "arn:aws:s3:::${var.bucket_name}/app-data/*"
          # ✅ Specific prefix only, not entire bucket
        ]
        Condition = {
          Bool = { "aws:SecureTransport" = "true" }  # HTTPS only
        }
      },
      {
        Effect = "Allow"
        Action = [
          "secretsmanager:GetSecretValue"
        ]
        Resource = [
          "arn:aws:secretsmanager:${var.region}:${var.account_id}:secret:${var.name}/*"
        ]
      },
      {
        Effect = "Allow"
        Action = [
          "ssm:GetParameter",
          "ssm:GetParameters"
        ]
        Resource = [
          "arn:aws:ssm:${var.region}:${var.account_id}:parameter/${var.name}/*"
        ]
      }
    ]
  })
}

resource "aws_iam_role_policy_attachment" "app_s3" {
  role       = aws_iam_role.app_role.name
  policy_arn = aws_iam_policy.app_s3_read.arn
}

resource "aws_iam_instance_profile" "app" {
  name = "${var.name}-app-profile"
  role = aws_iam_role.app_role.name
}

MFA Enforcement Policy

Enforce MFA for all human users — a must for compliance (CIS AWS Benchmark, SOC2).

hcl — IAM MFA enforcement
resource "aws_iam_policy" "require_mfa" {
  name        = "RequireMFA"
  description = "Forces users to use MFA. Allows only MFA setup actions without MFA."

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        # Allow MFA device management even without MFA
        Sid    = "AllowManageOwnMFA"
        Effect = "Allow"
        Action = [
          "iam:CreateVirtualMFADevice",
          "iam:EnableMFADevice",
          "iam:ListMFADevices"
        ]
        Resource = [
          "arn:aws:iam::*:mfa/&{aws:username}",
          "arn:aws:iam::*:user/&{aws:username}"
        ]
      },
      {
        # Deny EVERYTHING else if no MFA session
        Sid    = "DenyWithoutMFA"
        Effect = "Deny"
        NotAction = [
          "iam:CreateVirtualMFADevice",
          "iam:EnableMFADevice",
          "sts:GetSessionToken"
        ]
        Resource = "*"
        Condition = {
          BoolIfExists = {
            "aws:MultiFactorAuthPresent" = "false"
          }
        }
      }
    ]
  })
}

IRSA — IAM Roles for Service Accounts (EKS)

For EKS, pods should never share the node's IAM role. Each pod gets its own minimal role via IRSA (OIDC federation). This is the secure EKS pattern.

hcl — EKS IRSA
# OIDC provider for EKS cluster
data "tls_certificate" "eks" {
  url = aws_eks_cluster.main.identity[0].oidc[0].issuer
}

resource "aws_iam_openid_connect_provider" "eks" {
  client_id_list  = ["sts.amazonaws.com"]
  thumbprint_list = [data.tls_certificate.eks.certificates[0].sha1_fingerprint]
  url             = aws_eks_cluster.main.identity[0].oidc[0].issuer
}

# Role for a specific pod/service account
resource "aws_iam_role" "pod_s3" {
  name = "${var.name}-pod-s3-role"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Allow"
      Principal = {
        Federated = aws_iam_openid_connect_provider.eks.arn
      }
      Action = "sts:AssumeRoleWithWebIdentity"
      Condition = {
        StringEquals = {
          # Only THIS specific service account in THIS namespace can assume this role
          "${aws_iam_openid_connect_provider.eks.url}:sub" =
            "system:serviceaccount:${var.k8s_namespace}:${var.k8s_service_account}"
        }
      }
    }]
  })
}

Service Control Policies — AWS Organization Guard Rails

hcl — SCP: Deny actions outside approved regions
resource "aws_organizations_policy" "deny_unapproved_regions" {
  name        = "DenyUnapprovedRegions"
  description = "Prevent resource creation in non-approved regions"

  content = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Deny"
      Action    = "*"
      Resource  = "*"
      Condition = {
        StringNotEquals = {
          "aws:RequestedRegion" = ["us-east-1", "us-west-2"]
        }
      }
      NotPrincipal = {
        # Except global services: IAM, Route53, CloudFront
        Service = ["iam.amazonaws.com", "route53.amazonaws.com"]
      }
    }]
  })
}

resource "aws_organizations_policy" "deny_root_actions" {
  name    = "DenyRootAccountActions"
  content = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Deny"
      Action    = "*"
      Resource  = "*"
      Condition = {
        StringLike = { "aws:PrincipalArn" = "arn:aws:iam::*:root" }
      }
    }]
  })
}

from checkov.common.models.enums import CheckResult, CheckCategories
from checkov.terraform.checks.resource.base_resource_check import BaseResourceCheck
import re

class S3BucketNamingConvention(BaseResourceCheck):
    """
    Custom check: S3 buckets must follow naming convention
    Pattern: {env}-{team}-{purpose}-{account_short}
    e.g., prod-platform-logs-1234
    """

    def __init__(self):
        name = "S3BucketFollowsNamingConvention"
        id = "CKV_CUSTOM_1"
        supported_resources = ['aws_s3_bucket']
        categories = [CheckCategories.GENERAL_SECURITY]
        super().__init__(name=name, id=id,
                         categories=categories,
                         supported_resources=supported_resources)

    def scan_resource_conf(self, conf):
        bucket_name = conf.get("bucket", [""])
        if isinstance(bucket_name, list):
            bucket_name = bucket_name[0]

        pattern = r'^(dev|staging|prod)-[a-z]+-[a-z]+-\d{4}$'

        if re.match(pattern, str(bucket_name)):
            return CheckResult.PASSED
        return CheckResult.FAILED

scanner = S3BucketNamingConvention()

Auto-Remediation Lambda

When GuardDuty finds a compromised EC2 instance, automatically isolate it by moving it to a quarantine security group — buying you time to investigate.

python — lambda/guardduty_remediation.py
import boto3
import json
import os
import logging

logger = logging.getLogger()
logger.setLevel(logging.INFO)

ec2 = boto3.client('ec2')
sns = boto3.client('sns')

QUARANTINE_SG = os.environ['QUARANTINE_SG_ID']
ALERT_TOPIC   = os.environ['ALERT_TOPIC_ARN']

def lambda_handler(event, context):
    finding = event['detail']
    finding_type = finding['type']
    severity     = finding['severity']
    account_id   = finding['accountId']

    logger.info(f"GuardDuty finding: {finding_type} (severity: {severity})")

    resource = finding.get('resource', {})
    resource_type = resource.get('resourceType')

    # Handle compromised EC2 instance
    if resource_type == 'Instance':
        instance_id = resource['instanceDetails']['instanceId']
        isolate_instance(instance_id, finding_type, severity)

    # Handle compromised IAM credentials
    elif resource_type == 'AccessKey':
        access_key_id = resource['accessKeyDetails']['accessKeyId']
        username      = resource['accessKeyDetails']['userName']
        disable_access_key(username, access_key_id, severity)

def isolate_instance(instance_id, finding_type, severity):
    """Move instance to quarantine SG — cuts off all normal network access"""
    try:
        # Get current security groups
        response = ec2.describe_instances(InstanceIds=[instance_id])
        instance = response['Reservations'][0]['Instances'][0]
        current_sgs = [sg['GroupId'] for sg in instance['SecurityGroups']]

        # Replace ALL security groups with quarantine SG only
        ec2.modify_instance_attribute(
            InstanceId=instance_id,
            Groups=[QUARANTINE_SG]
        )

        logger.info(f"ISOLATED instance {instance_id}. Was in SGs: {current_sgs}")

        # Add forensic tag
        ec2.create_tags(
            Resources=[instance_id],
            Tags=[
                {'Key': 'SecurityStatus',   'Value': 'QUARANTINED'},
                {'Key': 'SecurityFinding',   'Value': finding_type},
                {'Key': 'OriginalSGs',       'Value': ','.join(current_sgs)},
            ]
        )

        sns.publish(
            TopicArn=ALERT_TOPIC,
            Subject=f"🚨 EC2 QUARANTINED: {instance_id}",
            Message=f"Instance {instance_id} quarantined.\nFinding: {finding_type}\nSeverity: {severity}\nOriginal SGs: {current_sgs}"
        )

    except Exception as e:
        logger.error(f"Failed to isolate {instance_id}: {e}")
        raise

def disable_access_key(username, access_key_id, severity):
    """Disable compromised IAM access key"""
    iam = boto3.client('iam')
    try:
        iam.update_access_key(
            UserName=username,
            AccessKeyId=access_key_id,
            Status='Inactive'
        )
        logger.info(f"DISABLED access key {access_key_id} for user {username}")
        sns.publish(
            TopicArn=ALERT_TOPIC,
            Subject=f"🚨 IAM KEY DISABLED: {username}",
            Message=f"Access key {access_key_id} for {username} has been disabled.\nSeverity: {severity}"
        )
    except Exception as e:
        logger.error(f"Failed to disable key {access_key_id}: {e}")

🧠 How Terraform Works (Mental Model)

Terraform has 3 core phases: Write (HCL code → describes desired state), Plan (terraform plan → compares desired vs actual, shows diff), Apply (terraform apply → makes AWS match desired state).

The state file (terraform.tfstate) is Terraform's memory — it tracks everything it has created. If you delete a resource from the state without deleting it from AWS, Terraform loses track. Never manually edit state.

Security implication: State files can contain secrets (RDS passwords, API keys). Always store in encrypted S3 with DynamoDB locking. Never commit to git.
